# Voice Command Classifier

A production-style keyword spotting system that classifies short audio clips
into voice commands using **MFCC-based feature extraction** and an **XGBoost**
classifier. No deep learning, no NLP — pure classical ML.

---

## Target Commands

| Label   | Description        |
|---------|--------------------|
| left    | "left"             |
| right   | "right"            |
| up      | "up"               |
| down    | "down"             |
| stop    | "stop"             |
| unknown | any other command  |

## Dataset

Download the **Speech Commands** dataset from Kaggle:

```
https://www.kaggle.com/datasets/neehakhatri/google-speech-commands
```

Extract the archive so the folder layout matches:

```
data/raw/speech_commands/
  left/    0a2b400e_nohash_0.wav  ...
  right/   0a9f9af7_nohash_0.wav  ...
  up/      ...
  down/    ...
  stop/    ...
  yes/     ...   ← becomes "unknown"
  no/      ...   ← becomes "unknown"
  ...
```

## Usage

### Step 1 — Prepare the dataset

Traverse the audio files, extract features, and save `features.csv`:

```bash
python src/prepare_dataset.py
```

Outputs:
- `data/processed/features.csv` — flat feature matrix (one row per audio file)
- `data/processed/dataset_metadata.json` — dataset statistics and config snapshot

Expected runtime: ~5–15 minutes depending on hardware.

---

### Step 2 — Train the model

Run GridSearchCV to tune XGBoost, then evaluate on validation and test sets:

```bash
python src/train.py
```

Outputs:
- `artifacts/model.joblib` — best sklearn Pipeline (scaler + XGBClassifier)
- `artifacts/label_encoder.joblib` — fitted LabelEncoder
- `artifacts/train_metadata.json` — best params, split sizes, classification reports
- `reports/confusion_matrix.png` — confusion matrix on the test set

Expected runtime: ~10–30 minutes (32 grid combinations × 3 folds).

---

### Step 3 — Predict a single file

```bash
python src/predict.py path/to/audio.wav
```

Example output:

## Feature Engineering

Each 1-second audio clip at 16 kHz is converted into an **88-dimensional**
fixed-length vector:

| Feature              | Dimensions | Stats   |
|----------------------|------------|---------|
| MFCC (13 coefs)      | 13         | mean+std |
| Delta-MFCC           | 13         | mean+std |
| Delta²-MFCC          | 13         | mean+std |
| Zero Crossing Rate   | 1          | mean+std |
| RMS Energy           | 1          | mean+std |
| Spectral Centroid    | 1          | mean+std |
| Spectral Bandwidth   | 1          | mean+std |
| Spectral Rolloff     | 1          | mean+std |
| **Total**            | **88**     |         |

## Configuration

All parameters are defined in `src/config.py`:

| Parameter          | Default | Description                        |
|--------------------|---------|------------------------------------|
| SAMPLE_RATE        | 16000   | Audio sample rate (Hz)             |
| DURATION           | 1.0     | Clip length (seconds)              |
| N_MFCC             | 13      | Number of MFCC coefficients        |
| RANDOM_STATE       | 42      | Reproducibility seed               |
| TEST_SPLIT         | 0.15    | Fraction held out for testing      |
| VAL_SPLIT          | 0.15    | Fraction held out for validation   |
| INCLUDE_UNKNOWN    | True    | Add an "unknown" catch-all class   |
| MAX_UNKNOWN_SAMPLES| 2000    | Cap on unknown-class samples       |

## Model

- **Pipeline**: `StandardScaler` → `XGBClassifier`
- **Tuning**: `GridSearchCV` (3-fold stratified CV, macro F1 scoring)
- **Serialisation**: `joblib`

GridSearch parameters explored:

| Parameter        | Values         |
|------------------|----------------|
| n_estimators     | 100, 200       |
| max_depth        | 4, 6           |
| learning_rate    | 0.05, 0.1      |
| subsample        | 0.8, 1.0       |
| colsample_bytree | 0.8, 1.0       |
